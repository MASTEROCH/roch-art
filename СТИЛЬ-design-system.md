# 🎨 ROCH Landing — Стилистический подход (Design System)

Фиксирует визуальный язык лендинга, чтобы продолжать разработку в VS Code без потери стиля.

---

## КОНЦЕПЦИЯ

**Liquid Glass (iOS 26) × Gamedev UI.** Тёмная «жидкое стекло» эстетика Apple, скрещённая с энергетикой игрового интерфейса: HUD-акценты, XP-бары, ринг-гейджи, ранги, реактивный маскот. Ощущение — дорогой продукт, сделанный человеком из геймдева.

Принцип: **фон живёт отдельно от контента** (глубина), стекло реально преломляет свет, всё дышит и реагирует на действие. Ноль внешних ассетов — всё нарисовано в CSS/SVG.

---

## ЦВЕТА (CSS-переменные)

```css
--bg:      #050706;   /* почти чёрный, основа */
--text:    rgba(255,255,255,.97);
--text2:   rgba(234,244,238,.72);   /* вторичный */
--text3:   rgba(234,244,238,.45);   /* приглушённый */
--lime:    #D4FF4F;   /* фирменный акцент ROCH */
--green:   #0F8A57;   /* градиентная пара к lime */
--lime-soft: rgba(212,255,79,.15);
--card:    rgba(21,29,24,.68);   /* база стекла ПОД градиентом */
```

**Правило акцента:** lime — только для важного (акцент-слово, CTA, активные состояния, цифры). Не заливать им площади. Градиент `lime → green` — для колец, баров, свечения.

**Ключевой градиент текста** (`.grad`): `#fff → lime → #7BE0A8 → #fff`, анимированный шиммер. Только на 1 акцент-слове в заголовке.

---

## ТИПОГРАФИКА

- Шрифт: **системный SF Pro** (`-apple-system`) → на iPhone рендерится родным Apple-шрифтом (половина «вау»). Fallback — **Inter**.
- Заголовки: `font-weight: 800–900`, `letter-spacing: -.04em` (плотный трекинг = премиум).
- H1: `clamp(38px, 10.5vw, 58px)`. Кинетический reveal по словам (маски + `translateY`).
- Body: 500, `line-height: 1.5`.
- Микро-лейблы: 700–800, `uppercase`, `letter-spacing: .1em+`.

---

## СТЕКЛО (главный примитив)

```css
.glass{
  background: var(--glass-hi), var(--card);  /* градиент-блик + тёмная база */
  backdrop-filter: blur(24px) saturate(170%);
  border: 1px solid rgba(255,255,255,.12);
  box-shadow: 0 26px 64px -14px rgba(0,0,0,.7),
              inset 0 1px 0 rgba(255,255,255,.18);   /* верхняя световая кромка */
}
```

**Критично для iOS:** всегда солидная тёмная база (`--card`) ПОД полупрозрачным градиентом. Если WebKit вырубит `backdrop-filter` — карточка остаётся тёмной и читаемой. Стекло = бонус, не несущая стена.

- Радиусы: карточки `32px`, средние блоки `18px`, пилюли `99px`.
- Спекулярный блик — псевдоэлемент `::before` с radial-gradient сверху-слева.

---

## ДВИЖЕНИЕ

- Easing: спринг `cubic-bezier(.34,1.45,.42,1)` для входов/нажатий, `cubic-bezier(.2,.7,.2,1)` для плавного.
- **Только `transform` и `opacity`** на крупных/анимируемых слоях. НИКОГДА `filter` или `mix-blend-mode` на fixed/полноэкранных слоях — кладёт iOS WebKit.
- Наборы: boot-каскад входа, шиммер градиента, sweep-блик по кнопкам, breathe-свечение CTA, blur-fade reveal при скролле, 3D-tilt карточек к пальцу, count-up цифр, ринг-гейдж заливка, лаймовые частицы вверх.
- `prefers-reduced-motion` уважается везде — анимации выключаются, контент остаётся.

---

## ФИРМЕННЫЕ ЭЛЕМЕНТЫ (авторские, не шаблон)

1. **Аудит-линза** — стеклянная лупа, которую водишь по «до», под ней «после». Пружинное следование за пальцем + автоблуждание в простое. Это услуга ROCH, ставшая интеракцией.
2. **Реактивный маскот** — SVG-бот в палитре ROCH, реагирует эмоциями на ответы квиза (радость/сомнение/шок) + пузыри реплик.
3. **Ринг-гейдж** результата квиза (SVG, заливка градиентом).
4. **Призрачные номера секций** 01–05 + гигантский водяной знак ROCH в футере.
5. **Наклонённая бегущая строка** между секциями.
6. **Живые миниатюры кейсов** — пульсирующие/мигающие скелетоны (пока нет реальных скриншотов).

---

## ФОН (не трогать структуру)

`.bg` — **fixed div** (не body-background: превью/webview его игнорируют). Внутри: чёрная заливка + 3 радиальных градиента (зелёный/лайм/тил) + анимированные орбы + частицы. Параллаксит от скролла. Это доказанно рендерится везде.

## СТРУКТУРА СЕКЦИЙ

```
Header (sticky, прячется вниз)
Hero (заголовок + лид + 3 стата + аудит-линза)
Ticker (бегущая строка)
01 Services (3 стеклокарточки, tilt)
02 Audit (маскот + квиз + ринг-гейдж)
Zero (нет продукта? ещё лучше)
03 Cases (4 живых кейса + автор)
04 Pricing (3 пакета, feat светится) + Payment (крипта/IP + копирование)
05 Process (4 шага)
Final CTA
Footer (водяной знак ROCH)
CTA-bar (плавающий, выезжает после hero)
```

---

## ПРАВИЛА КОНТЕНТА

- **Не перегружать.** Один блок = одна мысль. Понятно ребёнку и старику.
- Ценность > фичи. «Чтобы купили» вместо «красивый UI».
- Каждая секция ведёт к действию (Telegram / оплата).
- Цифры-доказательства на видном месте (700M+, 24ч).
